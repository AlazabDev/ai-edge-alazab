## 🔌 التكاملات المتاحة

### 1. Bot Gateway API
**الغرض**: نقطة نهاية موحدة لكل عمليات الصيانة والاستعلامات

**الإجراءات المدعومة** (17 إجراء):
- إنشاء طلبات صيانة
- فحص حالة الطلبات
- إدارة المستخدمين والتفضيلات
- إدارة الفنيين
- كاتالوج الخدمات
- عروض الأسعار

### 2. Maintenance Gateway
**الغرض**: إنشاء طلبات الصيانة من القنوات الخارجية

### 3. Query Gateway
**الغرض**: استعلامات مجمعة مع pagination

### 4. Daftra Integration (جاهز للتفعيل)
**الغرض**: المزامنة المحاسبية والفواتير الإلكترونية

## 🔄 سيناريوهات الاستخدام

### السيناريو 1: طلب صيانة متكامل

```
المستخدم → "أحتاج صيانة كهربائية في الإسكندرية"
         ↓
البوت    → يفهم النية، يطلب تفاصيل
         ↓
البوت    → ينادي createMaintenanceRequest()
         ↓
API      → ينشئ الطلب في النظام
         ↓
البوت    → يرسل رقم التتبع
         ↓
WhatsApp → إشعار فوري للعميل
```

### السيناريو 2: متابعة طلب

```
المستخدم → "ما حالة طلبي UF/MR/260502/0042"
         ↓
البوت    → ينادي checkRequestStatus()
         ↓
API      → يرجع الحالة الحالية
         ↓
البوت    → يقدم تحديث مفصل
```

### السيناريو 3: توصيات ذكية

```
المستخدم → "أريد ترميم شقتي"
         ↓
البوت    → يفهم نوع المشروع
         ↓
البوت    → ينادي brandNavigator()
         ↓
البوت    → يقترح الخدمات المناسبة
```
